# 任务链（Task Chain）模块产品需求文档

## 1. 概述

### 1.1 产品背景
任务链是机器人管理系统中的核心功能模块，用于定义和管理机器人执行任务的流程链条。通过任务链，管理员可以创建、编辑和监控复杂的任务流程，确保机器人能够按照预设的逻辑顺序执行各项操作。

### 1.2 产品目标
- 提供直观的任务链管理界面，支持任务链的创建、编辑、删除和查看
- 实现任务链的树形可视化展示，方便管理员理解任务流程
- 支持任务链的版本管理和历史记录
- 提供任务链执行状态的实时监控
- 允许任务链的导入导出，方便在不同环境间迁移

### 1.3 目标用户
- 系统管理员：负责创建和管理任务链
- 运维人员：监控任务链执行状态
- 开发人员：调试和优化任务链逻辑

## 2. 功能需求

### 2.1 任务链列表页面

#### 2.1.1 功能描述
- **任务链卡片列表**：以卡片形式展示所有任务链，每张卡片包含：
  - 任务链名称（tree_id）
  - 版本号
  - 任务链状态（启用/禁用）
  - 任务链快照（树形结构的缩略图）
  - 创建时间
  - 最后修改时间
- **搜索功能**：支持按任务链名称、版本号进行搜索
- **筛选功能**：支持按状态、创建时间等进行筛选
- **排序功能**：支持按创建时间、名称等排序
- **批量操作**：支持批量启用/禁用/删除任务链
- **创建按钮**：点击跳转到任务链创建页面

#### 2.1.2 页面流程
1. 用户进入任务链列表页面
2. 系统加载并展示所有任务链卡片
3. 用户可以通过搜索、筛选找到目标任务链
4. 用户点击卡片进入任务链详情页面
5. 用户点击创建按钮进入任务链创建页面

### 2.2 任务链详情页面

#### 2.2.1 功能描述
- **任务链基本信息**：展示任务链名称、版本号、创建时间、最后修改时间等信息
- **树形可视化**：以树形图形式展示任务链的完整结构，包括：
  - 节点类型（calc_equip, move, call_equip, plan, success, error, cancel）
  - 节点参数
  - 节点状态
  - 节点间的连接关系
- **节点编辑**：点击节点可编辑其参数和属性
- **节点操作**：支持添加、删除、移动节点
- **版本管理**：显示任务链的版本历史，支持版本回滚
- **执行历史**：展示任务链的执行历史记录
- **保存按钮**：保存对任务链的修改
- **返回按钮**：返回任务链列表页面

#### 2.2.2 页面流程
1. 用户从任务链列表页面点击进入详情页面
2. 系统加载并展示任务链的完整信息和树形结构
3. 用户可以查看、编辑任务链的各个节点
4. 用户可以添加、删除、移动节点来调整任务链结构
5. 用户点击保存按钮保存修改
6. 用户点击返回按钮回到任务链列表页面

### 2.3 任务链创建页面

#### 2.3.1 功能描述
- **基本信息填写**：
  - 任务链名称（tree_id）
  - 版本号
  - 描述信息
- **节点编辑器**：
  - 拖拽式节点添加
  - 节点参数配置
  - 节点连接关系设置
- **模板选择**：提供常用任务链模板供选择
- **导入功能**：支持从JSON文件导入任务链
- **保存按钮**：保存并创建任务链
- **取消按钮**：取消创建并返回列表页面

#### 2.3.2 页面流程
1. 用户从任务链列表页面点击创建按钮
2. 系统加载任务链创建页面
3. 用户填写基本信息
4. 用户通过拖拽或模板创建任务链结构
5. 用户配置各节点参数
6. 用户点击保存按钮创建任务链
7. 系统创建任务链并跳转到详情页面

## 3. 非功能需求

### 3.1 性能要求
- 任务链列表页面加载时间不超过2秒
- 任务链详情页面加载时间不超过3秒
- 树形结构渲染支持至少100个节点的复杂任务链

### 3.2 可用性要求
- 操作界面直观易用，减少学习成本
- 提供操作引导和帮助文档
- 支持快捷键操作，提高操作效率

### 3.3 安全性要求
- 任务链的创建和修改需要权限验证
- 重要操作需要确认机制
- 系统自动记录操作日志

### 3.4 可扩展性要求
- 支持自定义节点类型
- 支持插件化扩展功能
- 预留API接口供其他系统集成

## 4. 数据结构

### 4.1 任务链数据模型

```json
{
  "tree_id": "string",           // 任务链唯一标识
  "version": "string",           // 版本号
  "description": "string",        // 描述信息
  "status": "boolean",           // 状态（启用/禁用）
  "node_json": {                  // 节点结构
    "node_type": {               // 节点类型
      "param": {},               // 节点参数
      "replay_state": "number",  // 回放状态
      "next": [                  // 下一个节点
        {
          "state": "number",     // 状态（1:成功, 0:错误, 2:取消）
          "node_type": {}         // 节点类型
        }
      ]
    },
    "replay_state": "number",      // 回放状态
    "next": []                     // 下一个节点
  },
  "created_at": "timestamp",      // 创建时间
  "updated_at": "timestamp",      // 最后修改时间
  "created_by": "string",         // 创建人
  "updated_by": "string"          // 最后修改人
}
```

### 4.2 支持的节点类型
- **calc_equip**：设备计算节点（分配设备）
- **move**：移动节点（执行移动）
- **call_equip**：设备调用节点（调用设备层方法）
- **plan**：计划节点（规划）
- **success**：成功节点
- **error**：错误节点（异常上报）
- **cancel**：取消节点（取消任务）
- **release_equip**：释放机器人节点

### 4.3 设备层任务类型
- **chb_get_bag**：获取袋子
- **sing_charg**：充电

### 4.4 单机原子指令
- **dispatch_nav**：调度轨迹导航 (10011 2027)
- **docking**：停泊 (10004 2010)
- **exit_docking**：退出停泊 (10009 2024)
- **relative_move**：相对移动 (10005 2011)
- **hardware_control**：机器人硬件控制 (10012 2022)
- **hardware_task**：机器人操作任务 (10008 2025)
- **wait**：纯等待 (10010 2026)
- **free_nav**：自由导航 (10001 2006)
- **路径导航至地标**：10003

## 5. 界面设计

### 5.1 设计风格
- 采用与现有系统一致的设计风格
- 使用卡片式布局展示任务链列表
- 树形图采用层级缩进和连线展示
- 节点使用不同颜色区分类型：
  - 计算节点：蓝色
  - 移动节点：绿色
  - 设备调用节点：橙色
  - 计划节点：紫色
  - 成功节点：绿色
  - 错误节点：红色
  - 取消节点：灰色

### 5.2 页面布局
- **任务链列表页面**：
  - 顶部：搜索栏、筛选器、创建按钮
  - 中部：任务链卡片网格
  - 底部：分页控制

- **任务链详情页面**：
  - 顶部：任务链基本信息、操作按钮
  - 左侧：树形结构展示
  - 右侧：节点详情编辑
  - 底部：版本历史、执行历史

- **任务链创建页面**：
  - 顶部：基本信息填写
  - 中部：节点编辑器
  - 底部：操作按钮

## 6. 实现计划

### 6.1 开发阶段
1. **需求分析与设计**：完成PRD文档和UI设计
2. **前端开发**：实现任务链列表、详情、创建页面
3. **后端开发**：实现任务链数据存储和管理API
4. **集成测试**：测试功能完整性和性能
5. **部署上线**：部署到生产环境

### 6.2 关键技术点
- 前端：使用Tree.js或类似库实现树形结构可视化
- 后端：使用JSON存储任务链结构
- 数据传输：使用RESTful API进行前后端通信
- 版本管理：实现任务链的版本控制

## 7. 验收标准

### 7.1 功能验收
- 任务链列表页面能够正确展示所有任务链
- 任务链详情页面能够正确展示和编辑任务链结构
- 任务链创建页面能够成功创建新任务链
- 任务链的树形结构能够正确渲染
- 任务链的版本管理功能正常工作

### 7.2 性能验收
- 任务链列表页面加载时间不超过2秒
- 任务链详情页面加载时间不超过3秒
- 树形结构渲染支持至少100个节点的复杂任务链

### 7.3 兼容性验收
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 响应式设计，支持不同屏幕尺寸

## 8. 风险评估

### 8.1 潜在风险
- 复杂任务链的渲染性能问题
- 节点参数配置的复杂度
- 版本管理的冲突处理

### 8.2 风险缓解措施
- 实现树形结构的懒加载
- 提供节点参数的模板和默认值
- 实现版本冲突的检测和解决机制
