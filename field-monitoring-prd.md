# 场域监控模块产品需求文档

## 1. 概述

### 1.1 产品背景
场域监控是机器人管理系统中的核心可视化模块，用于实时展示机器人运行环境、位置状态、路径规划和任务执行情况。该模块将SLAM生成的原始地图数据转化为人类可读的地图界面，提供直观的场域监控和管理能力。

### 1.2 产品目标
- 将SLAM原始地图数据转化为人类可读的地图界面
- 实时展示机器人/AGV的位置、状态和运动轨迹
- 可视化展示地标、路径、虚拟墙、区域等场域元素
- 提供地图编辑和管理功能
- 支持多楼层、多区域的场域监控
- 实现地图与任务、设备管理的深度集成

### 1.3 目标用户
- 系统管理员：负责场域配置和地图管理
- 运维人员：监控机器人运行状态和场域情况
- 调度人员：查看机器人位置和任务执行进度
- 安全管理人员：监控场域安全状况和异常情况

### 1.4 成功指标
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 地图渲染性能 | <1秒 | 页面加载时间监控 |
| 实时更新延迟 | <500ms | WebSocket消息延迟监控 |
| 用户满意度 | >90% | 用户调研反馈 |
| 系统可用性 | 99.9% | 系统运行时间统计 |
| 地图元素识别准确率 | >95% | 用户测试验证 |

## 2. 功能需求

### 2.1 地图展示

#### 2.1.1 功能描述
- **地图渲染**：将SLAM原始地图数据转化为人类可读的地图界面
- **地图样式**：采用类似Google Maps或扫地机器人地图的样式
- **障碍物可视化**：将原始地图中的黑色点阵转化为清晰的障碍物（如墙壁）
- **区域划分**：使用不同颜色和图案区分不同区域
- **多楼层支持**：支持楼层切换和多层地图展示
- **地图缩放和平移**：支持地图的缩放、平移操作

#### 2.1.2 地图元素
| 元素类型 | 描述 | 可视化样式 |
|----------|------|------------|
| 障碍物 | 墙壁、柱子等不可通行区域 | 深灰色实线或填充区域 |
| 空白区域 | 可通行区域 | 白色或浅灰色背景 |
| 未扫描区域 | SLAM未扫描到的区域 | 浅灰色虚线或斜线填充 |
| 地标 | 关键位置标记点 | 彩色图标（如蓝色旗帜） |
| 路径 | 机器人行驶路径 | 彩色线条（如绿色） |
| 虚拟墙 | 软件定义的禁行区域 | 红色虚线或红色半透明区域 |
| 区域 | 功能区域划分 | 不同颜色的半透明区域 |
| 充电桩 | 机器人充电位置 | 黄色充电图标 |
| 电梯 | 电梯位置 | 电梯图标 |

#### 2.1.3 地图样式转换规则
- **原始SLAM地图**：
  - 空白：可通行区域
  - 黑色点阵：障碍物
  - 灰色：未扫描区域

- **转换后的地图**：
  - 可通行区域：白色背景 (#ffffff)
  - 障碍物：深灰色实线 (#2c3e50)，线条宽度2-3px
  - 未扫描区域：浅灰色斜线填充 (#e0e0e0)
  - 障碍物边缘：使用平滑算法消除点阵效果，形成连续的线条

#### 2.1.4 页面流程
1. 用户进入场域监控页面
2. 系统加载SLAM地图数据
3. 系统将原始地图数据转换为人类可读的地图样式
4. 系统渲染地图并叠加场域元素
5. 用户可以缩放、平移地图查看不同区域
6. 用户可以切换楼层查看不同楼层的地图

### 2.2 机器人/AGV展示

#### 2.2.1 功能描述
- **实时位置显示**：在地图上实时显示所有机器人/AGV的位置
- **状态标识**：使用不同颜色和图标标识机器人状态
- **运动轨迹**：显示机器人的运动轨迹和路径
- **机器人信息**：点击机器人图标显示详细信息
- **机器人筛选**：支持按状态、类型筛选机器人

#### 2.2.2 机器人状态可视化
| 状态 | 颜色 | 图标 | 说明 |
|------|------|------|------|
| 在线/空闲 | 绿色 (#27ae60) | 机器人图标 | 机器人在线且无任务 |
| 执行任务中 | 蓝色 (#3498db) | 机器人图标+任务标记 | 机器人正在执行任务 |
| 充电中 | 黄色 (#f39c12) | 机器人图标+充电标记 | 机器人正在充电 |
| 离线 | 灰色 (#95a5a6) | 机器人图标+离线标记 | 机器人离线 |
| 故障/异常 | 红色 (#e74c3c) | 机器人图标+警告标记 | 机器人出现故障 |

#### 2.2.3 机器人信息展示
点击机器人图标后显示的信息包括：
- 机器人ID和名称
- 当前状态
- 当前任务（如果有）
- 电量百分比
- 当前位置坐标
- 运行速度
- 最后更新时间

#### 2.2.4 页面流程
1. 系统实时接收机器人位置数据
2. 在地图上更新机器人图标位置
3. 根据机器人状态更新图标颜色和标记
4. 用户点击机器人图标查看详细信息
5. 用户可以使用筛选器查看特定状态的机器人

### 2.3 预测路径查看

#### 2.3.1 功能描述
- **预测路径展示**：在地图上显示机器人当前任务的预测行驶路径
- **路径规划**：基于当前任务和地图数据，计算机器人从当前位置到目标的最佳路径
- **实时更新**：当机器人位置或任务发生变化时，自动更新预测路径
- **路径详情**：显示预测路径的详细信息（距离、预计时间、途经点等）
- **路径对比**：支持显示实际行驶路径与预测路径的对比
- **多路径显示**：支持同时显示多个机器人的预测路径

#### 2.3.2 预测路径可视化
| 路径类型 | 颜色 | 线型 | 说明 |
|----------|------|------|------|
| 预测路径 | 蓝色 (#3498db) | 虚线 | 机器人预测行驶路径 |
| 实际路径 | 绿色 (#27ae60) | 实线 | 机器人实际行驶路径 |
| 偏差路径 | 红色 (#e74c3c) | 红色叉号 | 实际路径与预测路径的偏差 |
| 备选路径 | 橙色 (#e67e22) | 点线 | 备选预测路径 |

#### 2.3.3 预测路径信息结构
```json
{
  "robot_id": "string",        // 机器人ID
  "task_id": "string",         // 当前任务ID
  "predicted_path": {          // 预测路径
    "waypoints": [             // 途经点
      {
        "x": "number",         // X坐标
        "y": "number",         // Y坐标
        "floor": "number",     // 楼层
        "eta": "timestamp",    // 预计到达时间
        "landmark": "string"   // 关联地标（如果有）
      }
    ],
    "total_distance": "number", // 总距离（米）
    "estimated_duration": "number", // 预计时长（秒）
    "confidence": "number"     // 路径预测置信度（0-1）
  },
  "actual_path": {              // 实际路径
    "waypoints": [
      {
        "x": "number",
        "y": "number",
        "floor": "number",
        "timestamp": "timestamp"
      }
    ],
    "total_distance": "number",
    "actual_duration": "number"
  },
  "deviation": {               // 路径偏差
    "deviation_points": [],    // 偏差点列表
    "max_deviation": "number", // 最大偏差距离（米）
    "deviation_percentage": "number" // 偏差百分比
  }
}
```

#### 2.3.4 页面流程
1. 用户点击机器人图标查看详细信息
2. 系统显示机器人当前任务和状态
3. 用户点击"显示预测路径"按钮
4. 系统基于当前任务和地图数据计算预测路径
5. 系统在地图上以蓝色虚线显示预测路径
6. 用户可以查看预测路径的详细信息（距离、预计时间、途经点等）
7. 当机器人移动时，系统实时更新预测路径
8. 用户可以开启/关闭实际路径显示，与预测路径进行对比
9. 用户可以关闭预测路径显示

#### 2.3.6 高级功能
- **多路径对比**：支持同时显示多个机器人的预测路径，便于调度和冲突检测
- **路径优化建议**：当检测到路径冲突或低效路径时，提供优化建议
- **历史路径分析**：支持查看机器人历史任务的预测路径与实际路径对比
- **路径导出**：支持将预测路径导出为图片或数据文件
- **路径分享**：支持将预测路径分享给其他用户

- 支持至少1000个地图元素（地标、路径、区域等）

- 操作界面直观易用，减少学习成本
- 提供操作引导和帮助文档
- 支持快捷键操作，提高操作效率

- 地图编辑操作需要权限验证
- 重要操作需要确认机制
- 系统自动记录操作日志
- 支持地图数据的备份和恢复

### 3.4 可扩展性要求
- 支持自定义地图元素类型
- 支持插件化扩展功能
- 预留API接口供其他系统集成
- 支持地图数据的导入导出

## 8. 风险评估

### 8.1 潜在风险
- **地图样式转换复杂度**：SLAM原始地图转换为人类可读的地图样式算法复杂，可能影响性能
- **实时数据量大**：大量机器人的实时数据更新可能造成性能瓶颈
- **地图元素数量多**：大量地图元素可能影响渲染性能
- **编辑操作复杂**：复杂的地图编辑操作可能导致用户体验不佳
- **跨浏览器兼容性**：不同浏览器的Canvas/SVG渲染可能存在差异

### 8.2 风险缓解措施
- **地图样式转换**：
  - 实现高效的点阵平滑算法
  - 使用WebGL加速渲染
  - 实现地图分块加载和渲染
- **实时数据量大**：
  - 实现数据压缩和增量更新
  - 使用WebSocket进行高效通信
  - 实现数据节流和防抖
- **地图元素数量多**：
  - 实现地图元素的懒加载
  - 使用空间索引优化查询
  - 实现地图元素的层级渲染
- **编辑操作复杂**：
  - 提供清晰的操作引导
  - 实现撤销/重做功能
  - 提供操作确认机制
- **跨浏览器兼容性**：
  - 使用标准化的Canvas/SVG API
  - 进行充分的跨浏览器测试
  - 提供降级方案
